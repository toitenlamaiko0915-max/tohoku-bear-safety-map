## 2. アプリの目的

東北6県のクマ出没に関する公式情報への入口と、掲載可能な情報の概略位置を、地図と一覧で確認しやすくする。

本アプリは以下を目的としない。

* 公的機関の発表を代替すること
* リアルタイム性や完全性を保証すること
* 地域や場所の安全・危険を独自に断定すること
* 緊急通報や避難判断を代替すること

## 7. ユーザー導線

### 基本導線

1. `/`でサイトの位置づけ、最終更新日、注意文を確認する
2. `/map`で地図・一覧・フィルターを確認する
3. 各データの「公式情報ページへ」から一次情報を確認する
4. 必要に応じて`/official-links`、`/safety`、`/updates`を確認する

### 安全上の前提

* 本アプリだけで外出や避難の安全判断を完結しない
* 最新・詳細情報は公的機関の公式ページで確認する
* 緊急時は警察・消防・自治体への連絡を優先する

## 8. CSVデータ管理

### 8.1 保存場所

```text
public/data/bear_sightings.csv
```

### 8.2 現行スキーマ

```csv
id,prefecture,municipality,area,occurred_at,event_type,description,source_name,source_url,latitude,longitude,accuracy_m,status
```

| 項目 | 型・形式 | 運用上の扱い |
| --- | --- | --- |
| `id` | string | 一意。重複不可 |
| `prefecture` | string | 東北6県の県名 |
| `municipality` | string | 公式情報に基づく市町村名 |
| `area` | string | 詳細住所を避けた概略地域名 |
| `occurred_at` | `YYYY-MM-DD`を基本 | 公式情報で確認した発生日・目撃日 |
| `event_type` | enum | `目撃`、`痕跡`、`人身被害`、`捕獲済み` |
| `description` | string | 公式情報に記載された事実の短い要約 |
| `source_name` | string | 公式情報名 |
| `source_url` | URL | 公式ページ・マップ・資料のURL |
| `latitude` | number | 概略緯度。ピンポイント座標を避ける |
| `longitude` | number | 概略経度。ピンポイント座標を避ける |
| `accuracy_m` | number | 概略範囲。現在は主に1000〜3000m |
| `status` | enum | `published`、`draft`、`archived` |

### 8.3 現在のデータ状態

* 13列
* 48件
* 青森県13件、岩手県0件、宮城県5件、秋田県7件、山形県11件、福島県12件
* 現在の全行に座標があり、一覧48件・地図ピン48件を確認済み

### 8.4 入力ルール

* 公式情報だけを出典にする
* `source_url`を必須とし、確認できない情報は掲載しない
* 発生日・目撃日と、公式公開日・サイト更新日を混同しない
* 情報源にない内容を推測で補完しない
* 詳細住所、個人宅、学校、施設を特定できる粒度を避ける
* 座標は概略位置とし、`accuracy_m`を広めに設定する
* 掲載対象だけを`published`にする
* CSV更新時に列数、ID重複、日付形式、URLを確認する

## 10. 判定項目

### 10.1 表示対象

現在の実装は、CSV解析後に`status === "published"`の行だけを表示する。

未実装の検証：

* 必須項目の空欄検出
* 東北6県以外の県名排除
* `source_url`の存在・URL形式検証
* `occurred_at`の日付形式検証
* ID重複検出

### 10.2 地図表示

現在は緯度・経度を`Number`へ変換し、有限数である行だけを解析結果へ残す。

既知の課題：

* 空文字は`Number("") === 0`となるため、不正な座標として残る可能性がある
* 数値変換できない座標の行は一覧からも除外される
* 東北地方として妥当な座標範囲を検証していない

目標仕様は、座標がない行を地図から除外し、一覧には残すことである。

### 10.3 種別

* 有効値：`目撃`、`痕跡`、`人身被害`、`捕獲済み`
* 現在は不明な値を`目撃`へ正規化する
* 将来は不明値を検出し、推測で`目撃`に変更しない処理を検討する

### 10.4 鮮度

現在は`occurred_at`と閲覧時刻の差で以下に分類する。

| 表示 | 条件 |
| --- | --- |
| 24時間以内 | 24時間以下 |
| 3日以内 | 72時間以下 |
| 1週間以内 | 168時間以下 |
| 古い情報 | 168時間超 |

これは情報の新しさを示す表示であり、危険度ではない。将来の基準変更は別チケットで行う。

### 10.5 危険度

独自の危険度スコアや安全判定は実装しない。

使用可能な表現：

* 位置は概略表示です
* 公式情報ページをご確認ください
* 参考表示です
* 24時間以内、3日以内、1週間以内、古い情報

避ける表現：

* この場所は安全です
* この地域は危険です
* 必ずクマがいます
* 危険度○％

## 11. 基本ロジック

### 11.1 CSV読み込み

1. `/data/bear_sightings.csv`を`fetch`する
2. 独自CSVパーサーでヘッダーと行を配列化する
3. 現行型へ変換する
4. 緯度・経度が有限数の行だけを残す
5. `occurred_at`降順に並べる
6. `published`だけを表示対象にする

### 11.2 フィルター

* 初期状態は全県・全種別・全鮮度を選択
* 県、種別、鮮度のAND条件で絞り込む
* リセットで全選択へ戻す
* 最新情報が初期状態で隠れない構成

### 11.3 地図・一覧

* 同じ`filteredSightings`を使用する
* 各行にマーカーを作成し、`accuracy_m > 0`なら範囲円を表示する
* 一覧の表示件数制限はない
* 現在は座標欠損行を一覧だけに残せない

### 11.4 並び順

現在は`occurred_at`のみの降順。公開日、更新日、IDによる補助ソートは未実装。

### 11.5 最終更新日

現在はトップページに固定値を手動で記載する。CSVに`updated_at`列がないため自動算出しない。

更新時は次を混同しない。

* サイト最終更新日
* クマの発生日・目撃日
* 公式ページの公開日・更新日

## 12. エラー・例外時の扱い

| 状態 | 現在の実装 | 目標仕様 |
| --- | --- | --- |
| CSV取得失敗 | 地図下部に例外メッセージを表示 | 技術詳細を隠し、再試行・トップ・公式情報への導線を表示 |
| CSVが空 | 一覧0件になる | 「現在表示できる情報はありません」と公式情報への案内を表示 |
| フィルター結果0件 | 条件変更を促す | 現行表示を維持 |
| 座標欠損 | 0,0化または行全体除外の可能性 | 地図に出さず一覧へ表示 |
| 不正日付 | 明示的な除外・警告なし | 表示対象外とし、検証時に報告 |
| `source_url`欠損 | 明示的な除外なし | 掲載対象外 |
| 不明な種別 | `目撃`へ変換 | 入力エラーとして検出 |
| 不明なstatus | `draft`へ変換 | 入力エラーとして検出 |
| 外部リンク切れ | 実行時検知なし | 定期保守で確認し、公式代替URLへ更新 |

利用者へスタックトレース、内部パス、HTTP詳細などを表示しない。

## 14. 注意事項

### 情報源

* 県、市町村、警察、環境省等の公式情報を優先する
* SNS、ブログ、口コミ、伝聞だけの情報を掲載しない
* 公式情報で確認できない内容は推測しない

### 位置情報

* 詳細住所、個人宅、学校、施設を特定できる表示を避ける
* 公式情報が詳細でも、アプリでは概略地域・概略座標にする
* `accuracy_m`を位置の不確かさとして扱う

### 表現

* 本アプリは参考情報であることを明示する
* 最新性・完全性・安全性を保証しない
* 不安を過度にあおらず、独自判断を断定しない
* 公式情報への導線を維持する
