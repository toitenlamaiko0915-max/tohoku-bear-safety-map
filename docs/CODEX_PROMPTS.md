## このファイルの目的

Codexへ依頼する作業をチケット番号で管理し、対象範囲、禁止事項、検証結果、PRを追跡できるようにする。

詳細仕様は`docs/SPEC.md`、現在地は`docs/PROJECT_LOG.md`、検収結果は`docs/TEST_LOG.md`、公開手順は`docs/DEPLOY.md`を参照する。

---

## チケット管理ルール

### 形式

```text
TBM-CX-001｜作業名
```

* `TBM`：プロジェクトコード
* `CX`：Codex依頼
* `001`：3桁の連番

### 採番・運用

* 1つのCodex依頼につき1番号を付ける
* 次の番号は`PROJECT_LOG.md`と本ファイルの依頼履歴で確認する
* 仕様整理、データ更新、UI修正、検収を可能な限り分ける
* 正式依頼前の案は「未実施チケット候補」に置く
* 作業開始時、PR作成時、マージ時に状態を更新する
* PRをマージして検収が終わるまで「完了」にしない

### 状態

| 状態 | 意味 |
| --- | --- |
| 未着手 | 正式依頼前 |
| 依頼中 | Codexが作業中 |
| PR確認中 | PR作成済み・人間確認待ち |
| 修正依頼中 | PRへ追加修正中 |
| 検収中 | マージ前後の動作確認中 |
| 完了 | マージ・本番確認まで完了 |
| 保留 | 判断や外部条件待ち |
| 中止 | 実施しないことを決定 |

## チケットの粒度

### 推奨

* CSVスキーマと検証ルールの確定
* CSV最新情報の更新
* `/map`のエラー表示改善
* 最終更新日の表示改善
* 公式リンク疎通確認
* 1つの表示崩れ修正

### 避ける

* 仕様変更、CSV更新、全面デザイン変更を1件に混在させる
* 「いい感じに直す」など対象と完了条件が不明な依頼
* 公式情報を確認せずデータ追加まで任せる依頼
* 実装と本番マージを無確認で一括実行する依頼

## Codexへの禁止事項

### 共通

* 関係ないファイルや設定を変更しない
* 不要な依存関係を追加しない
* 認証、環境変数、秘密情報をコードへ埋め込まない
* `.env`、トークン、パスワードをコミットしない
* `main`への直接push、force push、履歴改変、無断マージをしない
* テストしていない項目を「確認済み」と報告しない

### 安全情報

* 非公式情報だけを根拠にCSVへ追加しない
* `source_url`を空欄にしない
* 情報源にない事実を推測しない
* 詳細住所、個人宅、学校、施設を特定できる情報を追加しない
* 「安全」「危険」「必ずいる」などを独自に断定しない
* 発生日・目撃日と、公式公開日・サイト更新日を混同しない

### UI・データ

* 依頼なしに全体デザインを変更しない
* スマホ表示、地図、一覧、公式リンクを壊さない
* CSVの列構造を移行手順なしに変更しない
* 0件やエラーを成功状態として隠さない

## 更新ルール

### 依頼前

* 最新番号を確認する
* 依頼を1つの目的に絞る
* 対象・対象外・禁止事項・完了条件を書く

### 依頼後

* 依頼履歴へ追加する
* PR作成・修正・検収・マージに合わせて状態を更新する
* 関連PRと未確認事項を記録する
* 完了後に`PROJECT_LOG.md`と`TEST_LOG.md`を更新する
